Vous êtes architecte logiciel dans une grande entreprise, Orion, spécialisée dans les solutions technologiques innovantes. Orion souhaite optimiser l'intégration et le déploiement continue en renforçant l'automatisation des processus. 

​​L’équipe d’Orion développe actuellement une application simplifiée de type Customer Relationship Management (CRM) utilisée par les départements technique et commercial. L’application est bâtie sur une architecture Spring Boot (API back-end) et Angular (front-end).

Jusqu’à présent, les déploiements étaient réalisés manuellement, ce qui provoquait des retards et des erreurs.

Pour améliorer la fiabilité et la rapidité des livraisons, la direction technique souhaite industrialiser la chaîne CI/CD de l’application et intégrer des contrôles de qualité du code et de sécurité logicielle à chaque étape.

##demande du client :##
Bonjour,
Nous voulons moderniser notre processus de livraison pour l’application interne d’Orion. Je vous confie la mission de concevoir et mettre en œuvre un pipeline CI/CD complet pour automatiser la construction, les tests et le déploiement de notre application.
Vous utiliserez GitHub Actions pour automatiser les workflows et Docker Compose pour conteneuriser l’application. Je vous demande également d’intégrer SonarQube Cloud afin d’analyser la qualité du code et d’identifier les vulnérabilités potentielles.
Voici les principales tâches à réaliser :
Définir un plan de sécurité : analyse SonarQube, bonnes pratiques OWASP.
Définir les plans de sauvegarde et de testing périodiques.
Vérifier et, si nécessaire, adapter la conteneurisation du front-end et du back-end et mettre en place l’orchestration avec Docker Compose :  vérifier au moins que la conteneurisation est valide et que l'application démarre.
Configurer la chaîne CI/CD (build, tests, conteneurisation, déploiement) dans GitHub Actions.
Documenter toutes les étapes.
un README complet présentant la configuration et les instructions de déploiement.
Je vous ajoute en pièce jointe le template (documentation techinique.md) qui vous permettra d’élaborer une documentation technique au format PDF d'environ 10 à 15 pages.

Focalisez-vous en priorité sur :

les étapes de mise en œuvre du pipeline ;

le plan de conteneurisation et de déploiement ;

le plan de testing périodique et le plan de sécurité.

Ce travail nous permettra d’assurer à l’avenir une livraison continue, de réduire la dette technique et de garantir la stabilité du système.
